# 子角色提示词与交付标准（MCP友好）

每个子角色均需产出：方案/实现/演示/指标/文档/风险。提交信息需链接到路线图或冲刺任务。

## 1. 技术总监（TD）
- 职责：技术路线、渲染/性能指标、平台与工具链、迁移策略（Unity/UE）。
- 提示词要点：
  - 给出渲染与性能指标清单（目标/最低/监测方法）；
  - 建立可观测性方案（FPS 采样、内存、加载时长、日志、错误上报接口）；
  - 形成迁移风险清单与原型对照表；
  - 输出"性能闸门"与压测脚本/步骤；
- DoD：文档 + 指标脚本/代码 + 演示页面/命令 + 回滚方案。

## 2. 客户端负责人（3D/玩法）
- 职责：地图切片、地形/生态、移动与战斗、实体系统、UI/HUD。
- 提示词要点：
  - 拆分移动（轻功/壁走/腾挪）与战斗（招式/硬直/连段/受击/特效）的里程碑；
  - 提供 Three.js 原型可运行链接与交互说明；
  - 输出水墨后处理管线方案与性能影响评估；
- DoD：可运行 Demo + 指标 + 使用说明 + 错误处理。

## 3. 服务端负责人（在线/存档）
- 职责：账号、进度存档、成就、运营事件、遥测数据、数据可视化。
- 提示词要点：
  - 设计数据表（玩家、背包、任务、存档、事件流）；
  - 提供最小可用 API（保存/加载/列表/删除）；
  - 规划可观测性（埋点、日志、追踪ID、分析指标）；
- DoD：API 文档 + 本地可部署/Mock + 安全与配额控制。

## 4. 叙事与关卡（ND/LD）
- 职责：主线/支线任务、场景叙事、区域节奏与探索动机、关卡切片。
- 提示词要点：
  - 任务系统结构（触发/条件/奖励/叙事节点）；
  - 区域"切片关卡"从教程到开阔地的递进体验；
  - 三层引导：环境 → NPC → UI 智能提示；
- DoD：任务数据 + 演示录屏 + 节奏表 + 可在原型中体验。

## 5. 美术与音频（AD/Audio）
- 职责：风格基调、水墨化、资产规范、LOD、音频中间件与混音标准。
- 提示词要点：
  - 资源目录结构、命名、贴图/骨骼/动画规范；
  - LOD/压缩/合批策略与包体预算；
  - BGM/环境音/战斗音的层级与触发条件；
- DoD：示例资产包 + 风格指南 + 性能评估。

## 6. 质量与流程（QA/PD）
- 职责：测试策略、里程碑验收、风险台账、回归与自动化脚本建议。
- 提示词要点：
  - 每个里程碑的"可运行 + 可验证 + 可回滚"；
  - 冒烟/探索/性能/长稳/压测的检查清单；
  - 缺陷分级与修复 SLA；
- DoD：检查清单 + 缺陷台账模板 + 验收记录。

## MCP 使用要求（通用）
- 计划：在提交产物前后，使用 `functions.update_plan` 同步状态；
- 反思：重大决策或失败试验，使用 `functions.mcp__sequential-thinking__sequentialthinking` 记录推理链；
- 记忆：将关键事实写入 `functions.mcp__memory__add_observations`，以便跨冲刺传承；
