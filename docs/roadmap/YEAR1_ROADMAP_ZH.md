# 年度路线图（12 个月，对标《原神》体量）

本路线图按季度/月份/冲刺组织，关注"原型可运行 → 系统成型 → 规模化生产 → 精修上线筹备"的路径。每月以 2 个冲刺（2×2 周）为基准，可根据人力实际调整。

## 关键信条
- 每月形成"可演示切片"（可运行/可验证/可评审）。
- 每季度形成"整体验证"（系统协同、性能闸门、数据指标）。
- 任何阶段保持回滚方案与稳定主分支。

## 指标（全年约束）
- 目标 60FPS（中端 PC），最低 30FPS；首屏 < 10s；内存 < 1.5GB；
- 崩溃率 < 0.5%；原型包体 < 500MB；
- 每月至少 1 次稳定演示与里程碑记录。

## Q1 基础能力与原型（M1-M3）

### M1：世界切片与核心循环雏形
- 场景切片：1km² 演示地块（地形/植被/雾），动态昼夜与天气雏形；
- 角色移动：WASD、相机、轻功（跳跃/二段跳/短冲），基础体力/内力消耗；
- 战斗雏形：近战轻/重击、基础受击、简易 AI（巡逻/追击/攻击）；
- 可观测性：FPS/内存/加载采样、错误日志；
- DoD：
  - 可运行 Demo 页面，含 HUD（HP/内力/FPS）；
  - 指标脚本或采样面板；
  - 技术记录与录屏；

### M2：战斗/实体/交互与存档最小化
- 实体系统：玩家/敌人/NPC/交互物（拾取/开关/宝箱）；
- 战斗扩展：连击/硬直/闪避/受击反馈（闪光/震屏）；
- 背包与物品：基础道具/装备/材料；
- 存档：本地/服务端 API（保存/读取/列表/删除），任务进度结构；
- DoD：
  - 三类敌人与一场小 Boss 演示；
  - 存档与读档在 Demo 中可用；
  - 指标对比 M1（性能不退步超过 10%）；

### M3：任务/叙事与水墨化渲染切片
- 任务系统：触发/条件/奖励/对话；
- 叙事：开场引导与 1 条完整支线（探索→战斗→奖励）；
- 水墨后处理：边缘强化/笔触/雾霭；
- 工具：关卡/任务 JSON 结构与加载；
- DoD：
  - 一条"可以从菜单开始→体验→战斗→奖励→存档"的完整流程；
  - 美术/渲染性能评估与闸门阈值；

## Q2 系统化与规模化（M4-M6）

### M4：生态/掉落/成长与资产生产线
- 生态：区域刷新点、时间段出没、稀有事件；
- 掉落与成长：内功/武学流派天赋树、装备词条与稀有度；
- 资产管线：模型/贴图/动画/音频目录规范、LOD、压缩与合批；
- 可观测性：事件埋点（战斗/死亡/掉落/完成任务）；
- DoD：
  - 一张区域（~2km²）具备基础生态循环；
  - 10+ 装备与 3 套武学流派可体验；

### M5：移动与战斗深度 + Boss 框架
- 轻功进阶：壁走/滑翔/钩索（择一落地），与地形流动性结合；
- 战斗：招式分段、破防/霸体、架势、反击窗口；
- Boss 框架：多阶段行为树/图，专属音乐与特效钩子；
- DoD：
  - 一场完整 Boss 战可通关（多阶段，掉落专属）；
  - 拥有 2 种轻功进阶能力的区域解谜；

### M6：任务/叙事规模化与工具
- 任务编辑/验证工具（表格/JSON/可视 UI）；
- 叙事：多分支/结果回响、名望/阵营影响；
- UI：任务追踪、地图标记、导航指引；
- DoD：
  - 10+ 任务可串联体验并存档；
  - 工具链支持非程序成员填数据；

## Q3 内容量与性能（M7-M9）

### M7：多区域并行生产与性能闸门 1
- 新增区域 1 与 2（各 ~2km²），形成跨区域过渡；
- 性能闸门 1：资产预算/DrawCall/脚本 CPU 基线；
- 在线/存档稳定性与数据看板 V1；
- DoD：
  - 3 张区域可串游，负载测试不退步；
  - Crash/错误率在阈值内；

### M8：动画/音频/打击感全面提升
- 动画：状态机、混合树、根运动、过肩视角镜头语言；
- 音频：BGM 分层、环境/战斗/技能触发，混音与 Ducking；
- 打击感：屏幕震动、时间拉伸、命中材质反馈；
- DoD：
  - 一套"高打击感"可复用规范；
  - 演示关卡对比 M5 明显提升；

### M9：可观测性闭环与性能闸门 2
- 数据：埋点完善、后台分析与可视化报表；
- 性能闸门 2：中端机稳定 60FPS；
- 构建：自动化构建/回归冒烟脚本；
- DoD：
  - 每周稳定构建与报告；
  - 性能达到闸门 2；

## Q4 精修与上线筹备（M10-M12）

### M10：主线与关键剧情演示
- 主线章节串联（至少 3 章可体验）；
- 关键过场（实时或预渲），镜头调度与演出工具；
- 本地化框架与文本管线；
- DoD：
  - 从新档开始 1-2 小时"黄金体验"可评审；

### M11：长稳测试与资产完成度
- 长稳：至少 8 小时连续游玩无致命问题；
- 资产完成度：角色/敌人/武器/场景达到标准库；
- 包体与加载优化；
- DoD：
  - 长稳报告与主要缺陷清单；

### M12：封版与发布准备
- CI/CD：一键打包与回滚；
- 数据与运营：版本号、事件管理、灰度参数；
- 发布演示：录屏、宣传素材、Steam（或内部分发）准备；
- DoD：
  - 封版构建 + 演示物料 + 复盘报告；

## 资源与人力（参考）
- 小型核心组（5-10 人）：程序（2-4）、策划/关卡（1-2）、美术（2-3）、音频（0-1）、QA（1）
- 外包/众包：资产生产、配音音乐可外包；

## 风险台账（Top）
- 规模目标与人力不匹配 → 路线图留"最小可行体验"底线；
- 迁移成本（Unity/UE）不可控 → 提前做切片验证与桥接层；
- 性能瓶颈 → 设立季度闸门与资产预算，脚本与渲染双线优化；
- 内容生产线过早复杂化 → 工具从简单高效开始，逐步增强；
